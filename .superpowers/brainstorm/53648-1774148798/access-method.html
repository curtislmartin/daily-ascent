How should the debug panel be accessed?

This affects how completely it can be hidden from production builds

A

Hidden Settings Section

A "Developer" section at the bottom of the Settings tab, compiled out with #if DEBUG. Works within existing navigation — no new UI patterns needed.

B

Long-Press Gesture

Long-press the Daily Ascent logo in TodayView to reveal the debug sheet. Invisible in production — no UI trace that it exists. Slightly harder to discover during testing.

C

Separate Debug Tab

A fourth "Debug" tab in the tab bar, compiled out entirely with #if DEBUG. Most visible and easiest to navigate during testing — but changes the tab bar layout while present.
